Tutorials

Introduction

We've worked hard to make Uranium as streamlined as possible, requiring the least possible effort from the user.

Part of this is ensuring that the widgets are easy to use and once you've learnt how to use one, the same methods are easily applied to other widgets.

As such, here we're going to go over an example of how to use widgets. We'll use the toggler as a model.

Tutorial: How to Implement A Toggler

We're going to start by making a toggler. What's a toggler? Well, it's a widget that has two components to it: a button and content. When we click on the button, something should appear. Here is what we're going to build (click on "Clothings"):

Clothings
  • Hat
  • Socks
  • Shoes
  • Gloves
  • Shirts

We click on the button ("Clothings") and the content (the list of clothes) appears.

Let's start with the basics: we need a div for "Clothings" and then an unordered list with the items in it.

<div>
  Clothings
<div>
<ul>
  <li> Hat </li>
  <li> Socks </li>
  <li> Shoes </li>
  <li> Gloves </li>
  <li> Shirts </li>
</ul>

We wrap the whole toggler in a div with the data-ur-set attribute set as "toggler".

Now we may have come across another problem. What if you can't wrap the whole thing in a div? Well, there's a trick around that - check out the explanation of sets vs ids here.

<div data-ur-set="toggler">
  <div>
    Clothings
  <div>
  <ul>
    <li> Hat </li>
    <li> Socks </li>
    <li> Shoes </li>
    <li> Gloves </li>
    <li> Shirts </li>
  </ul>
</div>

Next, we specify the button. We give the button the attribute "data-ur-toggler-component" and the value "button". (Want to know why the attributes are quite long? Click here.)

<div data-ur-set="toggler">
  <div data-ur-toggler-component="button">
    Clothings
  <div>
  <ul>
    <li> Hat </li>
    <li> Socks </li>
    <li> Shoes </li>
    <li> Gloves </li>
    <li> Shirts </li>
  </ul>
</div>

The final thing we need to do to the HTML is specify the content aspect of the toggler. The content in this case is the unordered list that we want to appear when the button is clicked. We give it the attribute data-ur-toggler-component (the same as for the button, if you remember) - but this time we use the value "content".

<div data-ur-set="toggler">
  <div data-ur-toggler-component="button">
    Clothings
  <div>
  <ul data-ur-toggler-component="content">
    <li> Hat </li>
    <li> Socks </li>
    <li> Shoes </li>
    <li> Gloves </li>
    <li> Shirts </li>
  </ul>
</div>

That's all we need to do with regards to HTML. Go back to the page, refresh. Try hitting the button... hm, nothing seems to happen! This is because we haven't changed the CSS yet. The javascript is doing its job, though! Let's see how.

Right-click on your widget and "inspect element". You'll see the button and the content divs have an extra attribute - data-ur-state. This is, by default, set to "disabled".

When the button is clicked, it changes to "enabled".

So, what we need to do is change the CSS of the page. It needs to be set so that the content does not display when data-ur-state="disabled", but to display when data-ur-state="enabled".

*[data-ur-toggler-component='content'] {
  display: none;
}
*[data-ur-toggler-component='content'][data-ur-state='enabled'] {
  display: block;
}

(This is actually what we call "lazy" CSS styling. What we're doing is only defining what happens when the data-ur-state is enabled - we don't say anything about its "disabled" state. Check out more about lazy vs proper styling here.)

The widgets within Uranium are pretty much only limited by your CSS imagination.
