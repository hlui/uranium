Responsive Table

Description

A responsive table displays only the number of columns that can fit on the screen size. So a smartphone will display two columns, but a browser will display all.

There is also some javascript that lets you control which columns are displayed via a "Display" menu on the right-hand side of the table.

Attributes

  • data-ur-set="flex-table" - add this attribute to a div that wraps the whole table
  • data-ur-flex-table-component="table" - add this attribute to the table tag
  • data-ur-flex-table-component="head" - add this attribute to the head tag of the table
  • data-ur-flex-table-component="body" - add this attribute to the body tag of the table
  • class="essential" - add this class to the header of a column that should be displayed (but still has the option of being hidden when unchecked in the menu
  • class="persist essential" - add this class to the header of a column that must always be displayed (and has no option of being hidden)
  • class="optional" - add this class to the header of a column that can be hidden when the screen size is too small

data-ur-set=

"flex-table"

data-ur-flex-table-component=

  • "table"
    • add this attribute to the table (the tag <table>)
    • multiple?: true
  • "head"
    • add this attribute to the table head (i.e. the tag <thead>)
    • multiple?: true
  • "body"
    • add this attribute to the table body (i.e. the tag <tbody>)
    • multiple?: true

class=

  • essential
    • add this attribute to a column header that should be displayed even in small screens
    • multiple?: true
    • attributes
      • optional: persist can be added to ensure the column is always displayed
  • essential
    • add this attribute to a column header that can be hidden on screen miniturization
    • multiple?: true

Instance

Each instance is available by set id under Ur.Widgets["tabs"]. Each instance exposes elements property -- raw components of the widget
