Togglers

Description:

A toggler is a widget that has two components - a button, and a set of contents. When you click the button, the states (of both the button and the contents) are toggled.

We could have an accordion-style toggler. For example, when you click on a category name, the list of contents is displayed.

How to Use:

Walk-Through of a Simple Example

We're going to go through the simplest scenario for a toggle. We have one button ("Clothings"). When we click on it, we want the content (containing the clothing options) to appear.

  • Socks
  • Shoes
  • Gloves

Let's start with the basics: we need a div for "Clothings" and then an unordered list with the items in it.

<div>
  Clothings
<div>
<ul>
  <li> Hat </li>
  <li> Socks </li>
  <li> Shoes </li>
  <li> Gloves </li>
  <li> Shirts </li>
</ul>

We wrap the whole toggler in a div with the data-ur-set attribute set as "toggler". (There is an alternative to using a "set" div - using IDs. Check it out here.)

<div data-ur-set="toggler">
  <div>
    Clothings
  <div>
  <ul>
    <li> Hat </li>
    <li> Socks </li>
    <li> Shoes </li>
    <li> Gloves </li>
    <li> Shirts </li>
  </ul>
</div>

Next, we specify the button. We give the button the attribute "data-ur-toggler-component" and the value "button".

<div data-ur-set="toggler">
  <div data-ur-toggler-component="button">
    Clothings
  <div>
  <ul>
    <li> Hat </li>
    <li> Socks </li>
    <li> Shoes </li>
    <li> Gloves </li>
    <li> Shirts </li>
  </ul>
</div>

The final thing we need to do to the HTML is specify the content aspect of the toggler. The content in this case is the unordered list that we want to appear when the button is clicked. We give it the attribute data-ur-toggler-component (the same as for the button, if you remember) - but this time we use the value "content".

<div data-ur-set="toggler">
  <div data-ur-toggler-component="button">
    Clothings
  <div>
  <ul data-ur-toggler-component="content">
    <li> Hat </li>
    <li> Socks </li>
    <li> Shoes </li>
    <li> Gloves </li>
    <li> Shirts </li>
  </ul>
</div>

That's all we need to do with regards to HTML. However, if you play around with your widget as it is, you'll notice not much is happening. This is because we haven't changed the CSS yet. The javascript is doing its job, though! Let's see how.

Right-click on your widget and "inspect element". You'll see the button and the content divs have an extra attribute - data-ur-state. This is, by default, set to "disabled".

When the button is clicked, it changes to "enabled".

So, what we need to do is change the CSS of the page. It needs to be set so that the content does not display when data-ur-state="disabled", but to display when data-ur-state="enabled".

*[data-ur-toggler-component='content'] {
  display: none;
}
*[data-ur-toggler-component='content'][data-ur-state='enabled'] {
  display: block;
}

(This is actually what we call "lazy" CSS styling. What we're doing is only defining what happens when the data-ur-state is enabled - we don't say anything about its "disabled" state - but more on that later.)

More Complex Examples:

Simple Example - with ID

What if we have two items that can't be wrapped in an overall div? In this case, we can't use the data-ur-set="toggler" attribute. This can be worked around by the data-ur-id trick. If the button and the content have a matching data-ur-id value, they will match up in their toggling. Simple!

<div data-ur-toggler-component='button' data-ur-id='MyFlatAccordionSocks'>
  Socks
</div>
<ul data-ur-toggler-component='content' data-ur-id='MyFlatAccordionSocks'>
  <li> Argyle </li>
  <li> Striped </li>
  <li> Solid </li>
  <li> Dr. Seuss </li>
</ul>

Popup -- with Lazy Styling

Now we're going to show two examples of popups. This one uses lazy styling, and the next will use "proper" styling. We aren't really using the 'disabled' data-ur-state --- we just style on the 'enabled' state. Check out the lazy CSS below.

div[name='Dialog'] *[data-ur-toggler-component='content'] {
  display: none;
  position: absolute;
  border-radius: 5px;
  background-color: #AAAAAA;
  width: 500px;
  height: 500px;
  left: 200px;
  bottom: 50px;
  text-align: center;
  padding-top: 30px;
}
div[name='Dialog'] *[data-ur-toggler-component='content'][data-ur-state='enabled'] {
  display: block;
}

Popup !

The pop-up appears way up the page. Not good. Let's try again.

Popup -- with Proper Styling

This time, we'll use proper styling for the popup. We add the data-ur-state attributes so that we can compartmentalize styles. Check out a proper CSS stying below.

.popup_content[data-ur-state='disabled'] {
  display: none;
  opacity: 0.5;
}
.popup_content[data-ur-state='enabled'] {
  display: inline-block;
  opacity: 1.0;
}

Popup !

Multiple Toggling Elements

Now we're getting to quite a complex example. The buttons are used to toggle between the two images. We start off by defining the button, giving it a data-ur-id and making its state disabled.

Then, we apply the same data-ur-id to the content and define the two content panes. However, we specify that one of the contents has a disabled state, and the other one has an enabled state. As they have opposing states, when the button is clicked they will remain the opposite of each other (i.e. the enabled image will become disabled, and vice versa).

<div data-ur-toggler-component='button' data-ur-id='MyToggler'
    class='buttons' data-ur-state='disabled'>
  <span class='popup_button'> Thing 1 </span>
  <span class='popup_button'> Thing 2 </span>
</div>
<div data-ur-toggler-component='content' data-ur-id='MyToggler'
    class='popup_content' data-ur-state='enabled'>
  <img src='images/popup.jpeg'>
</div>
<div data-ur-toggler-component='content' data-ur-id='MyToggler'
    class='popup_content' data-ur-state='disabled'>
  <img src='images/popup2.jpg'>
</div>
