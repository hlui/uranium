Geocode Tests / Examples

This widget allows you to reverse geocode a location. You can use this to populate forms using the user's location.

You may only use one of these widgets on a page, along with only one element per form. The elements you can populate are:

  • Street Address (only line 1, floor level is not present in the api)
  • City
  • Zip
  • State
  • Country
(view the source of an example for the attribute names)

The widget supports the use of either select elements or text input elements and should handle either gracefully. It will not support other types of elements.

The widget will fire onload if no button is present to activate the widget. If you include a button then the widget will only activate when clicked.

  • Widget fires onload
  • Widget fires on button click
  • Widget populates select elements
  • Widget populates text inputs

Things that might cause trouble when trying to make this work:

  1. Make sure your device is enabled to allow location services.
  2. Make sure your browser is enabled to allow location services.
  3. Running this as a local file may not work, you might have to run it from a server. Putting it in a public dropbox folder for testing is a good way to do this.
